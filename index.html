hamster.graphics¶

The hamster graphics library is a sprite-styled abstraction layer on top of cairo. Don’t worry if that does not make much sense right now. In short it allows you to draw things in gnome and python much faster and easier.

Sprite objects¶

class graphics.Sprite(x=0, y=0, opacity=1, visible=True, rotation=0, pivot_x=0, pivot_y=0, interactive=True, draggable=False)¶

The Sprite class is a basic display list building block: a display list node that can display graphics and can also contain children. Once you have created the sprite, use Scene’s add_child to add it to scene

add_child(*sprites)¶
add another sprite to this one to inherit position, rotation and opacity

Graphics class¶

class graphics.Graphics(context=None)¶

If context is given upon contruction, will perform drawing operations on context instantly. Otherwise queues up the drawing instructions and performs them in passed-in order when _draw is called with context.

Most of instructions are mapped to cairo functions by the same name. Where there are differences, documenation is provided.

See http://www.cairographics.org/documentation/pycairo/reference/context.html#class-context for detailed description of all the functions below.
